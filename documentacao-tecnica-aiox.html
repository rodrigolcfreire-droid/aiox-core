3 Pipeline

3.1 Pipeline de Ativação de Agentes (UAP)

Fonte: unified-activation-pipeline.js e 00-shared-activation-pipeline.md

1Entrada única: activate(agentId)
2Carga paralela de contexto (config de agente, sessão, projeto, git, permissão)
3Etapas sequenciais dependentes (greeting, tipo de sessão, estado de workflow)
4Montagem de enrichedContext
5Geração final do greeting/contexto via GreetingBuilder

Obs: generate-greeting.js atua como wrapper fino para retrocompatibilidade. Fallback em timeout e degradação graciosa quando alguma fonte falha.

3.2 Pipeline de Desenvolvimento (Story-Driven)

Fonte: constitution.md e .aiox-core/development/workflows/*.yaml

1Início obrigatório por story (docs/stories/)
2Implementação estritamente pelos acceptance criteria
3Atualização de checklist e file list
4Quality gates obrigatórios antes de concluir

Quality gates mandatórios:

npm run lint npm run typecheck npm test

6 Fluxos de Execução

6.1 Ativação de agente

1Usuário ativa agente
2UAP enriquece contexto com estado de sessão/projeto/permissões
3Agente inicia com greeting contextual e comandos do domínio

6.2 Desenvolvimento por story

1@sm / @po estruturam story e critérios
2@dev implementa e testa localmente
3@qa valida riscos, cobertura e aderência aos critérios
4@devops executa gates finais, push remoto, PR e release

6.3 Governança de git

1Trabalho e commits locais em branch de feature
2Push remoto somente via @devops
3PR/merge/release somente via @devops
4Regras reforçadas por constitution + hooks + definições de agente

6.4 Sincronização de ecossistema

1Atualização de agentes/rules/templates no core
2Execução de sync:ide para propagar para IDEs alvo
3Execução de validadores de sync/integration/parity
4Correção de drift e republicação de artefatos
